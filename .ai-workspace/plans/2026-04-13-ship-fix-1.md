# Ship-fix 1 — B1 bootstrap regex missed `async`

## Problem

PR #175 CI emitted `smoke-gate: active` with `HEAD has handleSmokeTest: no` even though the function exists in HEAD. Root cause: `scripts/smoke-gate-check.sh` used the regex `^export function handleSmokeTest\b`, but the real handler in `server/tools/evaluate.ts` is `export async function handleSmokeTest(...)`. The `async` keyword broke the match.

## Why local tests missed it

Tests 15 and 16 in `server/validation/smoke-gate-check.test.ts` built synthetic fixture repos with `export function handleSmokeTest() {}` — no `async`. The fixtures matched the regex, the regex matched the fixtures, and the test passed. But neither half was compared against the real `evaluate.ts` signature. Producer/consumer seam bug (P57-class — same pattern forge-plan is about to file to hive-mind-persist).

## Fix

1. **Regex update** (`scripts/smoke-gate-check.sh`): change to `^export \(async \)\{0,1\}function handleSmokeTest\b` (BRE syntax for an optional group).
2. **Test fixtures update** (`smoke-gate-check.test.ts`): use `export async function handleSmokeTest` in both test 15 and test 16 fixtures so they pin the async form.
3. **New sanity-check test** (`smoke-gate-check.test.ts`): read the real `server/tools/evaluate.ts` on disk and assert the JS-equivalent regex `/^export (?:async )?function handleSmokeTest\b/m` matches. This is the producer-consumer companion assertion that was missing.
