# Ship-fix #1 for PR #369 — ESLint unused-var in determinism test strip helper

## Context

CI build failed on both ubuntu-latest and windows-latest at the `Lint (eslint server/ — must exit 0)` step. Error:

```
server/lib/coordinator.test.ts
  430:26  error  '_elapsedMs' is assigned a value but never used  @typescript-eslint/no-unused-vars

✖ 1 problem (1 error, 0 warnings)
```

The executor used underscore-prefix convention (`const { elapsedMs: _elapsedMs, ...tbRest } = timeBudget`) for an intentionally-unused destructure. This repo's ESLint config does not configure `argsIgnorePattern`/`varsIgnorePattern: "^_"`, so the prefix doesn't exempt.

## Fix

Rewrite the `strip` helper to explicitly construct the deterministic-fields-only object — no unused variable, same semantic behavior, cleaner intent signal for future readers:

```typescript
const strip = (brief: typeof r1.brief) => ({
  ...brief,
  timeBudget: {
    maxTimeMs: brief.timeBudget.maxTimeMs,
    warningLevel: brief.timeBudget.warningLevel,
  },
});
```

Verified: `npm run lint` passes. `bash scripts/v033-1-acceptance.sh` still green (8/8 AC). No test semantics changed.
