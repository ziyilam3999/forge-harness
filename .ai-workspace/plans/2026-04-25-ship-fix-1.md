## Context

Stage 5 stateless reviewer on PR #428 returned PASS with 5 enhancements. One of them — E3 — is actually a CLAUDE.md privacy hard-rule violation that the reviewer classified too leniently. Reclassifying as a BUG.

The wrapper at `scripts/promo-pr-readme-acceptance.sh` contains the literal regulated token `UOB` at:
- **Line 134-135** (comment text): `"\`UOB\`" / "\`UOB Bank\`" / etc.`
- **Line 139** (fallback default): `tokens="UOB"`

The wrapper is part of PR #428's diff. PR #428 is public on GitHub. The literal token is visible in the diff. CLAUDE.md privacy hard-rule explicitly forbids this:

> "Pre-commit gate: before any commit / PR / push / release / public post / blog publish / chat output about the user's employer, grep staged or target content for the regulated token (case-insensitive — see the pinned privacy card for exact strings). Refuse to ship if matches appear in user-facing or public-bound files. Exception: rule-specification files intentionally carry the token to specify what's regulated — these are the per-project `feedback_no_employer_mention.md` memory file, the cross-session privacy card, and no others."

`scripts/promo-pr-readme-acceptance.sh` is NOT a rule-specification file. It is a public-bound script. The token must not appear there.

This is a **planner-side gap** — the original plan's AC-9 only grep-checks `README.md`, not the entire diff. The wrapper (introduced by this same PR) was outside AC-9's scope but inside the privacy rule's scope. The hard-rule won; AC-9 needs widening too.

## Goal (invariants when done)

- **G1.** The literal regulated token does NOT appear anywhere in `scripts/promo-pr-readme-acceptance.sh`.
- **G2.** The wrapper still satisfies its functional purpose: when the privacy card is present, it greps the README diff against the card's tokens; when the card is absent, it fails closed (rather than executing a hardcoded literal).
- **G3.** AC-9's verification scope widens from "the README diff" to "the entire PR diff" so the same class of bug can't recur in this PR or future ones.

## Binary AC

- [ ] **F1.** `grep -i 'UOB' scripts/promo-pr-readme-acceptance.sh | wc -l` returns `0`.
